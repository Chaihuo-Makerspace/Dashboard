# 响应式布局适配设计文档

**日期：** 2026-04-28  
**目标：** 仪表盘在 1280px ~ 1920px 范围内的笔记本屏幕上正常显示  
**方案：** Media Query 精细适配（方案 B）

---

## 背景

仪表盘当前为 1920×1080 大屏设计，在 1280×800 / 1366×768 / 1440×900 等笔记本分辨率下内容整体拥挤，面板内容被截断。

---

## 约束与决策

- **只改 CSS**，不修改 HTML 结构
- 三列布局在所有断点保持，不折叠
- 最小目标：1280px 宽度（以 1280×800 为测试基线）
- 1280px 时允许微调列比（中央地图列略宽），其余断点保持原比例
- 右侧面板已有 `transform: scale()` 逻辑，保留不动

---

## 断点策略

| 断点 | 目标设备 | 标签 |
|------|----------|------|
| ≥ 1920px | 外接大屏 / 桌面显示器 | 基准（现有样式） |
| ≤ 1680px | 15寸高分笔记本 | 轻量 |
| ≤ 1440px | MacBook Air M1/M2 | 中等 |
| ≤ 1280px | 13寸 MacBook / 主流 Windows 本 | 紧凑 |

---

## 各断点变更明细

### `#dashboard` 网格参数

| 属性 | 基准 | ≤1680px | ≤1440px | ≤1280px |
|------|------|---------|---------|---------|
| `grid-template-columns` | `3fr 5.5fr 3fr` | 不变 | 不变 | `2.8fr 6fr 2.8fr` |
| `gap` | `1.5rem` | `1.2rem` | `0.9rem` | `0.65rem` |
| `padding` | `1.5rem 1.5rem 0.5rem 1.5rem` | `1.2rem 1.2rem 0.4rem 1.2rem` | `0.9rem 0.9rem 0.35rem 0.9rem` | `0.65rem 0.65rem 0.3rem 0.65rem` |

### 面板内边距（覆盖 Tailwind `.p-4`）

| 断点 | padding |
|------|---------|
| ≤ 1440px | `0.75rem` |
| ≤ 1280px | `0.6rem` |

### `panel-title` 间距

| 属性 | 基准 | ≤1440px | ≤1280px |
|------|------|---------|---------|
| `margin-bottom` | `1rem` | `0.6rem` | `0.4rem` |
| `font-size` | `0.9rem` | `0.8rem` | `0.75rem` |
| `padding-bottom` | `0.5rem` | `0.4rem` | `0.3rem` |

### 大数字字体（温度、湿度、气压、风速）

| 元素 | 基准 | ≤1440px | ≤1280px |
|------|------|---------|---------|
| 温度/湿度（`.text-4xl`） | `2.25rem` | `1.9rem` | `1.6rem` |
| 气压/风速（`.text-3xl`） | `1.875rem` | `1.5rem` | `1.2rem` |

### 地图状态卡最小宽度

| 断点 | `min-width` |
|------|-------------|
| ≤ 1680px | `220px` |
| ≤ 1440px | `190px` |
| ≤ 1280px | `165px` |

---

## 实施范围

所有改动集中在 `index.html` 的 `<style>` 块末尾追加三组 `@media` 规则，覆盖现有样式。

**不需要改动：**
- HTML 结构
- JS 逻辑
- 右侧面板现有 `--panel-scale` 媒体查询（保留）
- ECharts 图表尺寸（由容器 flex 自动适应）

---

## 验收标准

1. 在浏览器中将窗口缩至 1280px 宽，所有面板内容可见，无横向滚动条
2. 1440px 下各面板内容不被截断
3. 1920px（原始尺寸）外观无变化
4. 右侧面板 scale 行为与改动前一致
